# AgroNesvizh — Astro + Netlify

Готовый сайт агроусадьбы в Несвиже: карточки номеров, страницы покоев, iCal-синхронизация с Суточно.ру, заявка в Telegram и винтажная дизайн-система.

## Что внутри

- Astro в hybrid-режиме.
- Netlify Adapter для API-эндпоинтов.
- Tailwind CSS.
- Content Collections вместо базы данных.
- `node-ical` для чтения занятых дат из `.ics` календарей Суточно.ру.
- `flatpickr` для календаря выбора дат.
- Telegram API для передачи заявок владельцу.

## Быстрый запуск локально

```bash
npm install
npm run dev
```

Сайт откроется на `http://localhost:4321`.

## Переменные окружения

Создайте `.env` по примеру `.env.example`:

```env
TELEGRAM_BOT_TOKEN=123456789:AA...
TELEGRAM_CHAT_ID=123456789
```

Без этих переменных форма откроется, но отправка заявки в Telegram вернёт ошибку.

## Деплой на Netlify

1. Загрузите проект в GitHub.
2. Создайте новый сайт в Netlify из этого репозитория.
3. Build command: `npm run build`.
4. Publish directory: `dist`.
5. В Netlify → Site configuration → Environment variables добавьте:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
6. Нажмите Deploy.

## Где редактировать номера

Файлы номеров лежат здесь:

```txt
src/content/rooms/
```

Каждый номер — отдельный Markdown-файл. В нём можно менять цену, описание, удобства, фото, ссылку Суточно.ру и iCal-ссылку.

## Где редактировать статьи

```txt
src/content/guides/
```

## Важная логика бронирования

Сайт не закрывает даты автоматически. Сценарий такой:

1. Гость выбирает номер и свободные даты.
2. Отправляет заявку через форму.
3. Заявка приходит владельцу в Telegram.
4. Владелец связывается с гостем.
5. После подтверждения вручную закрывает даты на Суточно.ру.

## Календарь занятости

API:

```txt
/api/availability/[slug]
```

Пример:

```txt
/api/availability/room-2
```

Эндпоинт читает iCal-ссылку из Markdown-файла номера, парсит будущие брони на 60 дней вперёд и возвращает массив занятых дат.

## Что стоит заменить перед боевым запуском

- Уточнить финальные цены в BYN.
- Проверить все iCal-ссылки Суточно.ру.
- Добавить юридически точную политику обработки персональных данных под владельца.
- При желании скачать фото локально и заменить внешние URL на `/images/rooms/...`.
